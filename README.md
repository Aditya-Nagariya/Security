## ğŸ”® DUAL-CORE ARCHITECTURE

**Aegis Security System** represents the evolution of open-source security dashboards. We are currently executing a strategic migration from a monolithic Python application to a high-performance, concurrent Rust architecture.

*   **Aegis RS (`aegis_rs/`)**: The new production core. Built with `tokio` for async execution and `iced` for a declarative, GPU-accelerated UI. It is designed for mission-critical Linux environments where performance and type safety are paramount.
*   **Aegis Py (`Aegis/`)**: The original rapid prototype. Built with `tkinter` and `psutil`. It serves as the functional specification and reference implementation.

## ğŸ¦€ Aegis RS (Production Core)

The new high-performance core.

### Key Features
*   **Memory Safety:** Guaranteed by Rust's ownership model.
*   **Concurrency:** Heavy security scans run on background `tokio` threads without freezing the UI.
*   **Modern UI:** Built with `iced` (Elm Architecture) for a reactive, glitch-free experience.
*   **Simulation Mode:** Develop on macOS/Windows with mock data, deploy on Linux with real tools.

### ğŸ› ï¸ Installation & Usage

**Prerequisites:** `cargo`, `lynis`, `ufw`, `clamav`

```bash
# 1. Install Dependencies (Debian/Ubuntu/Parrot)
sudo apt install build-essential lynis ufw clamav

# 2. Build Release Binary
cd aegis_rs
cargo build --release

# 3. Run (Root required for active scans)
sudo ./target/release/aegis_rs
```

## ğŸ Aegis Py (Legacy Prototype)

The original reference implementation.

### Key Features
*   **Rapid Prototyping:** Quick iteration using Python's dynamic typing.
*   **Tkinter UI:** Standard library GUI (no complex compilation).
*   **Direct Shell Execution:** Uses `subprocess` for system interaction.

### ğŸ› ï¸ Installation & Usage

```bash
# 1. Install System Tools
sudo apt install python3-tk lynis clamav rkhunter

# 2. Install Python Deps
pip3 install psutil

# 3. Run
sudo python3 security.py
```

## ğŸ”§ Architecture Comparison

We are moving from **Imperative (OOP)** to **Functional (MVU)** architecture.

| Feature | Aegis Py (Legacy) | Aegis RS (Modern) |
| :--- | :--- | :--- |
| **Language** | Python 3.10+ | Rust 1.70+ |
| **Concurrency** | `threading` (GIL restricted) | `tokio` (Async/Await) |
| **State Mgmt** | Shared Mutable State | Message Passing (Actor Model) |
| **GUI** | Tkinter (Imperative) | Iced (Declarative/Reactive) |
| **Performance** | Interpreter Overhead | Native Machine Code |
| **Safety** | Runtime Errors Possible | Compile-Time Guarantees |

2.  **Port to Rust:** Implement the logic in `aegis_rs/src/core/`.
3.  **Wire UI:** Add the view logic in `aegis_rs/src/ui/`.
4.  **Submit PR:** Open a Pull Request with the tag `[RUST-MIGRATION]`.
